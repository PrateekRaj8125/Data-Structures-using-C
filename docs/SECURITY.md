# Security Policy

## 📢 Reporting a Vulnerability

We take security and code quality seriously.

If you discover a vulnerability, bug, or unexpected behavior in this repository, please follow responsible disclosure:

### 🚫 Do NOT

- Publicly disclose the issue before it is reviewed and resolved

### ✅ Do

1. Open a [GitHub Security Advisory](https://github.com/PrateekRaj8125/Data-Structures-using-C/security/advisories)  
    OR
2. Report via email: <prateek.raj.indt@gmail.com>

---

## 📄 What to Include in Your Report

Please provide as much detail as possible:

- Description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Possible impact
- Suggested fix (if any)

---

## 🔔 What to Expect

- ⏱ Acknowledgment within **48 hours**
- 🔍 Investigation within **7 days**
- 🛠 Fix released if confirmed
- 🙌 Credit provided (optional)

---

## 🎯 Scope

This repository is primarily educational.  
Security concerns may include:

- Unsafe memory handling
- Undefined behavior in C programs
- Logical vulnerabilities in algorithms

---

✅ Responsible reporting helps maintain a safe and reliable learning resource for everyone.
